# 競馬データの取得（スクレイピング）について

本システム（keiba-research-mvp）は、競馬予想の研究に使用するための「箱」です。
デフォルトではダミーデータを生成して動作確認を行いますが、**実際に機能させるためには、過去のレースデータや最新の出走表を外部から取得（スクレイピング等）してデータベースに投入する必要があります。**

## データの主な入手元
日本の競馬において、データを取得する一般的な方法は以下の通りです。

### 1. netkeiba.com (推奨)
最もポピュラーな競馬情報ポータルサイトです。多くの個人開発者が機械学習・データ分析用途にスクレイピングを行っています。
- **取得可能データ**: 過去数十年分のレース結果、払い戻し、血統、出馬表など。
- **注意点**: サーバーに負荷をかけないよう、必ずリクエスト間隔（1秒〜2秒以上）を空けてアクセスしてください。

### 2. JRA-VAN DataLab.
JRA公式のデータサービス（有料）。
- **メリット**: 公式の正確なデータ、オッズのリアルタイム更新などが取得可能。
- **注意点**: Windows専用のSDK経由でのアクセスとなるため、Node.jsやクラウド上のサーバーから直接叩くことは困難です。通常はWindowsマシン上でPython等を用いて中継サーバーを立てる必要があります。

### 3. Yahoo!競馬 / スポーツナビ
- シンプルな出馬表や結果であれば取得可能ですが、詳細なラップタイムや過去の細かいデータはnetkeibaに劣ります。

## スクレイピングパイプラインの構築
実際のデータを使って本システムを運用するための、推奨されるデータの流し込み方は以下の通りです。

1. **スクレイパーの作成 (Node.js/Python)**
   - `puppeteer`, `playwright`, または `cheerio/axios` 等を用いて、対象サイトからHTMLを取得し、JSON形式にパースします。
   - `scripts/scraper-sample.ts` に、Cheerioを用いた簡単なスクレイピングのひな形を用意しました。
2. **データの整形**
   - 取得したデータを、本システムのSupabaseテーブル（`race`, `horse`, `runner`, `odds_snapshot`）の形式にマッピングします。
3. **Supabase への投入**
   - `scripts/seed_supabase.ts` で実装したように、SupabaseのJavaScriptクライアントを用いてInsert（またはUpsert）を実行します。

## プロジェクトへの統合手順
本レポジトリ内にスクレイパーを統合する場合は、以下のように進めてください：

1. パッケージのインストール： `npm install axios cheerio`
2. `scripts/` ディレクトリ配下にスクレイパーを構築。
3. GitHub Actions や Vercel Cron ジョブで、週末に最新の出走表を自動スクレイピングしてDBを更新するように設定。

※ **免責事項**: スクレイピング対象サイトの利用規約（robots.txtなど）を必ず確認し、負荷をかけない常識的な頻度でのアクセスを遵守してください。
